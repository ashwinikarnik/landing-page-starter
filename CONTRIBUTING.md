# Contributing Guide

## Branching

- Create feature branches from `main`.
- Use concise branch names, e.g. `feat/loan-input-validation`.

## Commit Messages

Use clear, scoped messages.

Examples:

- `feat: add loan amount input component`
- `fix: handle invalid interest rate values`
- `docs: update setup instructions`

## Pull Requests

Before opening a PR:

1. Rebase on latest `main`.
2. Run lint, tests, and build locally.
3. Add screenshots for UI changes.
4. Include test notes in PR description.

## Code Quality

- Keep components focused and reusable.
- Prefer explicit types and validation.
- Avoid introducing dead code or unused dependencies.

## Documentation

Update docs when behavior, setup, or configuration changes.
